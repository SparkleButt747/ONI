01 · Design philosophy
01.A
Graphic Realism
Objects are real but rendered with graphic clarity. No photorealism, no pure abstraction. Every surface communicates its function through visual language — labels, stripes, colour-coding. Reality, amplified.
01.B
Utility as aesthetics
Industrial signage, hazard markings, stencil type, warning colours — these are not decorations. They ARE the design. ONI's UI borrows this: every chrome is functional, every stripe earns its place.
01.C
Saturated clarity
High-chroma accent colours on near-black backgrounds. No gradients diluting the signal. No blending. Flat planes of colour separated by hard edges. The palette is a system, not decoration.
01.D
Machine consciousness
ONI is an onboard intelligence. The UI should feel like looking at a system's self-representation — diagnostic, precise, slightly alien. Ghost in the Shell HUDs, not Apple glass morphism.
01.E
Retrofuturism
Monospace terminals, CRT glitch artifacts, scan-line textures — 90s sci-fi futures as imagined from now. Not nostalgia: an aesthetic with ideological weight. Technology that has been used, worn, maintained.
01.F
No comfort phrasing
Mirrors ONI's personality. No rounded-corner softness, no pastel reassurance. Sharp cuts, hard borders, direct type. The design communicates the same way ONI speaks: terse, precise, honest.

04 · Motion + interaction
Scan entry
duration: 180ms · easing: steps(8)
Elements enter via scanline reveal — not fade. Left-to-right or top-to-bottom pixel exposure. Feels like data being loaded, not a CSS transition.
Glitch pulse
trigger: error state · duration: 120ms × 3
On error: element X-shifts ±3px three times in rapid succession with coral border flash. No bounce, no spring. Hard, digital, wrong.
Border pulse
trigger: active task · 2s loop · border-color only
Running tasks pulse their border between oni-border and acc-amber. No opacity change, no scale. The border is the activity indicator.
Type stream
ONI responses: character-by-character
ONI output streams one character at a time in the REPL. Not a typewriter affectation — reinforces the machine-printing aesthetic and matches actual SSE streaming.
Cursor
7×13px block · steps(1) blink · 1s
Block cursor, not underline. Amber fill. Binary blink — on/off, no fade. Identical across all input contexts. Never changes shape.
State transition
duration: 0ms (instant) or 150ms max
State changes are near-instant. ONI is a machine. Machines don't ease in. Reserve 150ms only for content that needs a moment to assemble (multi-panel layouts).
05 · UI component language
RUNNING
Status tags
Monospace, uppercase, 9px, 1px border matching text colour. No background fill. No border-radius. States: RUNNING (amber) · BLOCKED (coral) · DONE (lime) · ERROR (coral solid bg) · IDLE (muted).
Progress bars
5px height, zero border-radius, no animation. Fill colour encodes meaning: amber = active, lime = complete, coral = overbudget. Track is oni-border. No percentage label on bar — label above in mono.
[ tool ] read_file
Tool call badges
Format: `[ plugin ] tool_name arg` on one line. Brackets in muted, tool_name in cyan, arg in oni-muted. Built-in vs plugin distinguished by prefix only. Never a visual card — always inline mono text.
Hazard dividers
Amber/black diagonal stripe, 6px height, full width. Used to separate major sections. Also appears on destructive action warnings, rate-limit alerts, budget overruns. Direct reference to Marathon's industrial safety signage.
ONI
Section headers
Barlow Condensed 900, uppercase, white, 22px. Left border accent in section colour (coral = errors, cyan = info, amber = active, lime = success). No background. Hard left edge. Never centred.

07 · Do / don't
do
  • Hard borders — 1px solid, never box-shadow
  • Flat colour fills — no gradients
  • Block cursor, amber, binary blink
  • Barlow Condensed for all labels, uppercase
  • Share Tech Mono for all terminal / tool output
  • Hazard stripes to denote danger, limits, warnings
  • Left-aligned everything — never centred text in UI
  • Status always visible: token count, burn rate, task count
  • Colour encodes meaning — never decoration
  • Near-black background (#0a0a09) — not true black
don't
  • Border-radius > 0 on any structural element
  • Glassmorphism, frosted panels, backdrop-filter
  • Drop shadows or box-shadow for decoration
  • Gradient fills between accent colours
  • Emoji anywhere in the UI
  • Inter, Roboto, or system UI fonts
  • Animated spinners — use border pulse instead
  • Easing curves > 150ms
  • Tooltips — information lives inline or not at all
  • Centred headers or hero text
